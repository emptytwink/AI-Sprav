# Тестовый архив для ИИ-сборщика справочника

Проект: **Автоматизированный комплекс КСД-01 «Вектор»**  
Версия документации: **2.4.1**  
Дата: **2026-05-12**

Этот архив специально подготовлен для проверки функции автоматической сборки справочника из загруженной документации. Документы искусственные, но имитируют реальный комплект: инструкции, регламенты, руководства, таблицы, схемы, API, резервное копирование, безопасность, эксплуатация и техническое обслуживание.

## Рекомендуемые настройки

```env
OLLAMA_BASE_URL=http://192.168.6.39:11434
OLLAMA_MODEL=qwen3:8b
AI_MAX_DOCS=50
AI_MAX_CHARS_PER_DOC=4000
AI_OLLAMA_TIMEOUT=1200
```

## Что должен сделать ИИ

ИИ должен предложить структуру справочника примерно из таких разделов:

1. Общие сведения
2. Архитектура и состав комплекса
3. Требования к среде
4. Монтаж и подключение
5. Пусконаладка
6. Эксплуатация
7. Техническое обслуживание
8. Диагностика и аварии
9. Безопасность
10. Обновление ПО
11. Резервное копирование
12. Интеграции и API
13. Чертежи и схемы
14. Комплектность и ЗИП
15. Формы журналов и актов

## Контрольные файлы

- `control_expected_result/expected_ai_structure.json` — ожидаемая логика группировки.
- `control_expected_result/document_index.csv` — индекс всех файлов архива.

## Важная проверка

Файлы из папки `16_Смешанные_документы_для_классификации` специально сделаны неоднозначными. Хорошая ИИ-сборка должна учитывать не только имя папки, но и смысл документа.
